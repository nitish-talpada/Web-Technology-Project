Tips for a useful review

Short, focused feedback helps us reproduce and fix issues fast. Include steps, expected vs actual behaviour, and the platform/browser used.

Before you submit

  1. Complete Name and Rating. In Comments list steps to reproduce and expected result.
  2. Attach screenshots/logs only when helpful (recommended ≤5MB). Name files clearly (e.g. bug-login-2025-08-13.png).
  3. For feature requests explain who benefits and why.
  4. Don't include passwords or sensitive data. Provide email only if you want follow-up.
  5. This demo is client-side only — add server-side validation and secure storage before production.
